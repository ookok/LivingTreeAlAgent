### Phase 1-3: 语义索引 (`core/semantic_index.py`)

**核心架构 - 虚拟文件系统 (VFS)：**
```
压缩上下文
    │
    ▼
┌────────────────────────────────────────────────────┐
│              VirtualFileSystem (VFS)               │
├────────────────────────────────────────────────────┤
│  /context/                                         │
│  ├── /docs/           (文档节点)                    │
│  │   ├── section_1    (Markdown 分块)              │
│  │   └── section_2                              │
│  ├── /code/           (代码节点)                   │
│  │   ├── python/                                  │
│  │   │   └── block_1 (Python 代码块)               │
│  │   └── javascript/                               │
│  │       └── block_2 (JS 代码块)                   │
│  └── /mixed/          (混合内容)                   │
├────────────────────────────────────────────────────┤
│  索引层:                                           │
│  ├── tag_index:     标签 → 块ID                    │
│  ├── keyword_index: 关键词 → 块ID                   │
```

**核心能力：**

4. **语义搜索** - 基于意图的自然语言查询
5. **懒加载** - 按需加载，减少内存占用

**核心类：**
- `SemanticChunk` - 语义块数据结构
- `VirtualFileSystem` - 虚拟文件系统
- `SemanticIndexer` - 索引构建器
- `LazySemanticLoader` - 懒加载器
- `create_semantic_index()` - 便捷函数
- `quick_search()` - 快速搜索

**性能指标：**

**新增组件**：

- 自适应 TTL 调整（高频命中 → TTL 延长，最多3倍）
- 容量管理（内存/磁盘配额、淘汰阈值控制）

**新增接口**：
```python
cache.get_ttl_for_intent(intent, query)  # 获取意图对应TTL

```

##### 1.2 意图分类增强 (`core/agent_chat_enhancer.py`)

**新增分析功能**：
- 回答稳定性分析（高稳定性：翻译/摘要/知识查询；低稳定性：闲聊/调试）

- base_value × confidence_factor
- 翻译(0.9)、摘要(0.8)、知识查询(0.7)、代码生成(0.7)
