### 阶段A：核心自我进化 (P0) ✅

#### 1. IntentClarifier 自适应意图澄清
- **文件**: `client/src/business/hermes_agent/intent_clarifier.py`
- **核心原则**: ❌ 不预置固定的澄清问题列表 → ✅ 从交互中学习澄清策略
- **关键类**:
  - `AdaptiveClarifier`: 自适应意图澄清器
  - `ClarificationStrategy`: 澄清策略（从交互中学习）
  - `ClarificationResult`: 澄清结果
- **学习机制**:
  - 首次遇到新任务类型 → 使用LLM生成初始策略
  - 记录每次澄清的成功率
  - 成功率 < 40% 且使用次数 ≥ 3 → 重新学习策略
  - 持久化到 `~/.living.tree/clarifier_strategies.json`
- **LLM调用**: 通过 `GlobalModelRouter` (`ModelCapability.REASONING`)

#### 2. IntentRecognizer 意图识别器
- **文件**: `client/src/business/hermes_agent/intent_recognizer.py`
- **核心原则**: ❌ 不预置固定的意图分类规则 → ✅ 使用LLM动态识别
- **关键类**:
  - `Intent`: 意图数据结构（动态字段）
  - `IntentRecognizer`: 意图识别器
- **学习机制**:
  - 先尝试匹配已学习的模式（关键词匹配）
  - 未匹配 → 使用LLM识别
  - 置信度 > 70% → 学习新模式
  - 持久化到 `~/.living.tree/intent_patterns.json`

#### 3. ProgressiveUIRenderer 渐进式UI渲染
- **文件**: `client/src/business/hermes_agent/progressive_ui_renderer.py`
- **核心原则**: ❌ 不预置固定的渲染顺序 → ✅ 从用户交互中学习组件优先级
- **关键类**:
  - `ProgressiveUIRenderer`: 渐进式UI渲染器
  - `UIRenderState`: UI渲染状态（L1-L4）
  - `RenderPriority`: 渲染优先级（从交互中学习）
- **渲染时间线**:
  - T+0ms → L1: 骨架（立即返回）
  - T+100ms → L2: 核心结构
  - T+300ms → L3: 完整内容
  - T+500ms → L4: 丰富交互（按需）
- **学习机制**:
  - 记录用户查看时间、点击率、完成率
  - 动态计算优先级分数
  - 持久化到 `~/.living.tree/render_priorities.json`

#### 4. CogneeMemoryAdapter 记忆增强
- **文件**: `client/src/business/cognee_memory/cognee_memory_adapter.py`
- **核心原则**: ❌ 不预置固定的记忆结构 → ✅ 自动从文本中提取实体和关系
- **关键类**:
  - `CogneeMemoryAdapter`: 记忆增强适配器
  - `Memory`: 记忆数据结构（动态结构）
  - `RecallResult`: 召回结果
- **API实现**:
  - `remember(text, session_id)` → 存储记忆
  - `recall(query, session_id)` → 召回记忆
  - `forget(session_id, memory_id)` → 遗忘记忆
  - `improve(feedback)` → 从反馈中改进
- **存储**:
  - 知识图谱（`KnowledgeGraphBuilder`）
  - 向量数据库（`VectorStore`）
  - 持久化到 `~/.living.tree/cognee_memories.json`
- **混合排序**: 向量相似度 + 图谱关联度 + 访问频率 + 近期性 + 重要性分数

### 阶段B：技能与解析进化 (P1) ✅

#### 1. VibeSkillBuilder Vibe驱动技能构建
- **文件**: `client/src/business/skill_evolution/vibe_skill_builder.py`
- **核心原则**: ❌ 不预置固定的技能模板 → ✅ 从自然语言描述中自动生成技能
- **关键类**:
  - `VibeSkillBuilder`: Vibe驱动技能构建器
  - `Skill`: 技能数据结构（动态结构）
- **构建流程**:
  1. 分析技能需求（使用LLM）
  2. 生成技能结构（动态生成）
  3. 从示例中学习（如果提供）
  4. 验证并修复技能
  5. 注册到系统（生成SKILL.md）
- **学习机制**:
  - 记录技能使用次数和成功率
  - 从用户反馈中改进技能
  - 持久化到 `~/.living.tree/learned_skills.json`

#### 2. UniversalParser 通用文档解析
- **文件**: `client/src/business/bilingual_doc/universal_parser.py`
- **核心原则**: ❌ 不预置各文档类型的解析器 → ✅ 使用通用解析器 + 自适应学习
- **关键类**:
  - `UniversalParser`: 通用文档解析器
  - `DocumentStructure`: 文档结构（动态识别）
  - `ParsedResult`: 解析结果
- **解析流程**:
  1. 检测文档类型（匹配已学习类型 → LLM识别 → 学习新类型）
  2. 获取/学习结构模式
  3. 应用模式解析
  4. 返回结构化结果
- **学习机制**:
  - 学习文档类型的关键词和结构模式
  - 保存示例文档（前500字）
  - 从用户反馈中重新学习
  - 持久化到 `~/.living.tree/parser_patterns.json`

- 支持跨会话持续进化


### 阶段C：高级进化 (P2) ✅

#### 1. AdaptiveLearningLoop 自适应学习循环
- **文件**: `client/src/business/hermes_agent/adaptive_learning_loop.py`
- **核心原则**: ❌ 不预置固定的学习规则 → ✅ 从任务执行结果中学习
- **关键类**:
  - `AdaptiveLearningLoop`: 自适应学习循环器
  - `LearningSample`: 学习样本
  - `Policy`: 策略（从学习中优化）
  - `LearningMode`: 学习模式（监督/强化/无监督）
- **学习循环**:
  1. 评估结果质量（用户反馈优先 → LLM评估兜底）
  2. 记录学习样本
  3. 如果质量低（<60%），生成变体
  4. 测试变体，记录结果
  5. 更新策略（动态调整参数）
- **持久化**: `~/.living.tree/adaptive_learning.json`

#### 2. MultiAgentOrchestrator 多Agent编排
- **文件**: `client/src/business/hermes_agent/multi_agent_orchestrator.py`
- **核心原则**: ❌ 不预置固定的Agent协作流程 → ✅ 从任务中动态分解子任务
- **关键类**:
  - `MultiAgentOrchestrator`: 多Agent编排器
  - `Agent`: Agent定义（动态性能优化）
  - `SubTask`: 子任务
  - `AgentRole`: Agent角色（协调者/专业者/审核者/整合者）
- **协作流程**:
  1. 分解任务到子任务（匹配已学习模式 → LLM分解 → 学习新模式）
  2. 分配给专业Agent（从任务-Agent映射中学习）
  3. 并行执行（考虑依赖关系）
  4. 整合结果（使用LLM）
  5. 学习优化（更新协作模式）
- **持久化**: `~/.living.tree/multi_agent_orch.json`
