### 🔍 参考 RAG 最佳实践改进（P0-P2，2026-04-28 新增）

> 分析来源：Wang 等人（2024）《检索增强生成系统最佳实践》
> 综合匹配度：约 40%（混合搜索 85%，其他环节 10-50%）
> 文章作者：Louis-François Bouchard（Towards AI 联合创始人兼 CTO）

#### P0（立即借鉴）

- [ ] **添加查询分类机制**
  - 在 HermesAgent 中添加查询分类机制
  - 训练二元分类器（sufficient / insufficient）
  - 判断查询是否需要进行检索
  - 节省计算资源，提高响应速度
  - 参考文件：`client/src/business/hermes_agent/query_classifier.py`（新建）

- [ ] **添加重新排序（Reranking）机制**
  - 在 FusionRAG 中添加 reranking 机制
  - 使用 monoT5 或 RankLLaMA 对检索结果重新排序
  - 确保最相关的文档优先传递给生成器
  - 参考文件：`client/src/business/fusion_rag/reranker.py`（新建）

- [ ] **添加查询转换（Query Transformation）机制**
  - 在 FusionRAG 中添加查询转换机制
  - 支持查询重写、查询分解
  - 提高检索准确性
  - 参考文件：`client/src/business/fusion_rag/query_transformer.py`（新建）

#### P1（近期实施）

- [ ] **优化数据分块策略**
  - 优化 document_parser 的分块策略
  - 支持 256-512 token 块大小
  - 支持 small2big 策略、滑动窗口
  - 参考文件：`client/src/business/bilingual_doc/document_parser.py`

- [ ] **添加文档摘要（Summarization）机制**
  - 在 FusionRAG 中添加文档摘要机制
  - 使用 Recomp 或类似工具进行文档摘要
  - 降低 token 占用成本
  - 参考文件：`client/src/business/fusion_rag/document_summarizer.py`（新建）

- [ ] **优化嵌入模型**
  - 评估当前嵌入模型性能
  - 考虑切换到 FlagEmbedding LLM 或 Cohere
  - 参考文件：`client/src/infrastructure/model/embedding_client.py`

#### P2（可选）

- [ ] **添加文档重新打包（Document Repacking）机制**
  - 在 FusionRAG 中添加文档重新打包机制
  - 使用"反向"方法（相关性升序排列）
  - 参考文件：`client/src/business/fusion_rag/document_repacker.py`（新建）

- [ ] **添加 RAG 生成器微调能力**
  - 在 SelfEvolutionEngine 中添加 RAG 生成器微调能力
  - 使用相关文档和随机文档的混合进行微调
  - 参考文件：`client/src/business/self_evolution/rag_generator_finetuner.py`（新建）

- [ ] **添加多模态检索能力**
  - 在 FusionRAG 中添加多模态检索能力
  - 支持文本→图像、图像→文本检索
  - 参考文件：`client/src/business/fusion_rag/multi_modal_retriever.py`（新建）

## 🔍 参考 refly 改进（P0-P2，2026-04-28 新增）

> 分析来源：[refly-ai/refly](https://github.com/refly-ai/refly)（7.3k ⭐，开源智能体技能构建平台）

### 📊 基本信息
- **开发方**：refly-ai 团队
- **定位**：首个开源智能体技能构建平台（Vibe 驱动 + 确定性执行）
- **Star**：7.3k
- **协议**：Apache 2.0（附加 ReflyAI 开源许可证限制）
- **语言**：TypeScript 95.7%

| 功能维度 | refly | LivingTreeAI Agent | 匹配度 |
|---------|--------|-------------------|---------|

### 🏗️ 架构设计对比

#### **refly 架构（分层 + Monorepo）**
`
工具接入层（输入层）
├── 3000+ 第三方原生工具
├── 兼容 MCP 协议
└── 私有连接器（企业内部系统）

核心能力层
├── 构建模块：Vibe 驱动（自然语言 → Refly DSL）
└── 运行模块：有状态可干预运行时（确定性执行）

部署输出层（统一执行层）
├── REST API
├── 消息平台 Webhook（飞书/Slack/企微）
├── AI 编码工具插件（Claude Code/Cursor）
├── MCP 服务器
└── 低代码平台后端

治理层
├── 统一技能注册中心
├── 版本管控、团队空间协作
├── 审计日志、权限管理
└── 跨团队技能共享
`

#### **LivingTreeAI Agent 架构**
`
智能体层 (HermesAgent / EIAgent)
    ↓
统一工具层 (ToolRegistry + BaseTool)
    ↓
工具实现层 (20+ 工具模块)
    ↓
基础设施层 (GlobalModelRouter + PRISM + KnowledgeGraph)
`

**架构差异**：
- ✅ **相似点**：都支持 MCP 集成、技能构建
- 🔴 **差异点**：refly 有 Vibe 驱动技能构建、确定性执行保障、企业级治理

### 💡 可借鉴的技术点（按优先级排序）

#### **P0（立即借鉴，影响最大）**

##### **1️⃣ Vibe 驱动技能构建（最重要亮点）**

**refly 做法**：
- 用自然语言描述业务需求
- 由 Copilot 自动生成可复用技能
- 无需手动编排节点或编写大量样板代码
- 3 分钟内将企业 SOP 转化为可投产的智能体技能

**本项目可借鉴**：
`
当前：SkillEvolution 支持技能进化，但缺少自然语言生成技能能力
借鉴后：
  - 在 SkillEvolution 中添加 Vibe 驱动技能构建能力
  - 用户用自然语言描述需求 → 自动生成 SKILL.md
  - 无需手动编写技能文件
  - 降低技能创建门槛，提高技能复用性
`

**落地模块**：
- client/src/business/skill_evolution/（skill_evolution_engine.py，添加 Vibe 驱动构建）
- client/src/business/hermes_agent/（ibe_skill_builder.py，新建，Vibe 驱动技能构建器）

- 支持执行过程中暂停、审计、重新引导逻辑
- 保证 100% 操作合规
- 支持失败自动恢复、执行中热修复工作流

**本项目可借鉴**：
`
当前：工具执行缺少确定性保障，执行过程不可干预
借鉴后：
  - 在 ToolRegistry 中添加有状态可干预运行时
  - 支持执行过程中暂停、审计、重新引导逻辑
  - 保证操作合规性
  - 支持失败自动恢复、执行中热修复
`

**落地模块**：
