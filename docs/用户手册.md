## 1. 快速开始

### 1.1 启动应用

**Windows:**
```bash
# 运行客户端
run.bat client

# 运行中继服务器
run.bat relay
```

**Linux/macOS:**
```bash
python main.py client      # 启动桌面客户端
python main.py relay       # 启动中继服务器
python main.py all        # 启动所有服务
```

### 1.2 访问 Web 管理界面

启动中继服务器后，访问:
- 首页: http://localhost:8766/
- 管理控制台: http://localhost:8766/web
- API 状态: http://localhost:8766/api/health

### 1.3 首次运行

首次运行会自动:
1. 检测硬件环境
2. 初始化数据库
3. 显示主界面

### 3.1 首页

访问 http://localhost:8766/ 查看:
- 系统架构图
- 核心功能介绍

### 3.2 管理控制台

访问 http://localhost:8766/web 查看:

**数据概览:**
- 活跃客户端数
- 累计补丁数
- 累计报告数
- 覆盖周数

**进化趋势:**
- 按周查看模块热度
- TOP 10 模块排名

**中继配置:**
- 服务器地址/端口
- 最大客户端数
- 中继功能开关

**参数分发:**
- 重试超时
- 最大重试次数
- 心跳间隔
- 批处理大小

### 3.3 API 接口

**健康检查:**
```bash
curl http://localhost:8766/api/health
```

**配置同步:**
```bash
# 推送配置
curl -X POST http://localhost:8766/api/config/sync \
  -H "Content-Type: application/json" \
  -d '{"user_token": "xxx", "config_key": "app", "config_data": {}}'

# 拉取配置
curl "http://localhost:8766/api/config/sync?user_token=xxx"
```

**统计数据:**
```bash
curl http://localhost:8766/api/stats/overview
curl http://localhost:8766/api/stats/trends?weeks=8
```

## 常见问题

### Q1: 模型加载失败？

检查:
1. Ollama 服务是否运行
2. 模型文件是否完整
3. 硬件是否满足要求

### Q2: P2P 连接不稳定？

检查:
1. 网络防火墙设置
2. NAT 类型是否支持穿透
3. 中继服务器是否可用

### Q3: 如何导出数据？

在设置中:
1. 打开配置管理
2. 选择导出格式 (JSON/ZIP)
3. 指定保存路径

### Q4: Web 管理界面无法访问？

1. 确认中继服务器已启动
2. 检查端口 8766 是否被占用
3. 确认防火墙允许访问
