# Skill: 中继服务器配置

> 当用户配置或优化P2P网络连接时触发

## 触发条件
- 用户点击"连接中继服务器"
- P2P节点无法发现邻居
- 用户手动指定中继服务器

## 引导流程

### 步骤1：检查当前状态
```python
# 检查中继配置
p2p_get_relay_status()
```

### 步骤2：显示可用服务器
默认服务器列表：
- 139.199.124.242:8888 (主服务器)
- 139.199.124.242:8889 (备份)

### 步骤3：连接或更换服务器
```python
# 连接中继
p2p_connect_relay(relay_address="139.199.124.242:8888")
```

### 步骤4：验证连接
```python
# 验证连接状态
p2p_get_status()
```

## 故障排除

### 问题：连接超时
- 检查网络连通性
- 尝试备用服务器
- 检查防火墙设置

### 问题：节点数过少
- 等待Gossip协议扩散（最多30秒）
- 手动触发连接
```python
p2p_broadcast(message="新节点加入", message_type="config_sync")
```

## 性能优化
- 推荐最大邻居数：50
- 推荐连接池大小：10

## 成功标准
- `p2p_get_peers` 返回邻居数 > 0
- `p2p_get_relay_status` 所有服务器可达
