## 1. 重构背景与目标

### 1.1 当前挑战

| 挑战 | 描述 |
|-----|------|
| **代码冗余** | 多版本迭代导致代码混乱，存在未集成模块 |
| **架构割裂** | 业务逻辑分散，缺乏统一的生命系统视角 |
| **认知局限** | 当前基于"预测下一个字"范式，缺乏世界模型能力 |
| **进化能力弱** | 缺乏系统化的自进化机制 |
| **模块耦合** | 各模块间依赖复杂，难以独立演进 |

### 1.2 重构目标

**核心愿景**: 将LivingTreeAI从"统计概率生成器"蜕变为"具备世界认知、物理交互与自我演化能力的数字生命体"

| 目标 | 描述 |
|-----|------|
| **世界模型能力** | 实现 Next-State Prediction，学习物理规律与因果关系 |
| **生物机理驱动** | 复刻生物神经网络结构，产生自主智能 |
| **自进化架构** | 实现 Harness 架构，动态压缩上下文，经验沉淀为代码 |
| **统一感官大脑** | 单一模型统一多模态感知与推理闭环 |
| **代码清晰** | 消除冗余，建立清晰的模块边界 |

### 1.3 前沿理论融入

| 理论 | 应用位置 | 核心价值 |
|-----|---------|---------|
| **世界模型** | `world_model/` | 预测世界下一状态，实现"脑内预演" |
| **主动推理** | `life_engine/` | 自由能最小化，驱动自主行为 |
| **生物神经网络** | `cell_framework/` | 1:1复刻生物神经连接结构 |
| **Harness架构** | `evolution/` | Agent = Model + Harness，持续进化 |
| **统一多模态** | `perception/` | 打破多模型拼凑，统一感知推理 |

## 2. 核心架构原则

| 原则 | 说明 | 实施方式 |
|-----|------|---------|
| **生命中心** | 一切设计围绕数字生命体展开 | 以 `LivingSystem` 为核心，各模块为细胞组件 |
| **主动推理** | 系统主动预测、感知、更新信念 | 实现自由能最小化循环 |
| **自进化** | 系统能够自我优化和演进 | 经验沉淀、代码生成、自然选择 |
| **模块自治** | 每个模块具备独立生存能力 | 细胞架构，松耦合设计 |
| **可观测性** | 完整的状态监控和洞察能力 | 内置仪表盘和反思机制 |
| **无冗余** | 删除未使用代码，保持简洁 | 代码审查，功能验证 |

**进化循环**:

```python
async def evolution_cycle():
    """自主进化主循环"""
    while True:
        # 1. 收集经验
        experiences = await collect_recent_experiences()
        
        # 2. 评估性能
        performance = evaluate_system_performance()
        
        # 3. 生成变异
        mutations = generate_mutations(experiences, performance)
        
        # 4. 自然选择
        survivors = natural_selection(mutations)
        
        # 5. 应用优化
        apply_optimizations(survivors)
        
        # 6. 休眠等待
        await asyncio.sleep(EVOLUTION_INTERVAL)
```

## 5. 任务链架构

```
用户请求 → 意图识别 → 任务分解 → 资源评估 → 细胞组装 → 执行监控 → 结果反馈 → 学习沉淀
```

### 5.2 任务状态机

```
Pending → Queued → Processing → Completed/Failed → Archived
     ↓      ↓          ↓              ↓
   重试   取消      暂停/继续      重试/归档
```

|-----|------|---------|
| **串行执行** | 任务按顺序执行 | 有依赖关系的任务 |
| **并行执行** | 任务同时执行 | 独立任务集合 |
| **条件执行** | 根据条件决定路径 | 分支逻辑 |
