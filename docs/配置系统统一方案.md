## 一、现状问题

### 1.1 配置管理分散

项目存在 **7个独立的配置管理器**，功能重叠，难以维护：

| 模块 | 文件位置 | 大小 | 职责 |
|------|----------|------|------|
| **UnifiedConfig** | `core/config/unified_config.py` | 28KB | 全局配置（最完善） |
| **ConfigManager** | `core/config_manager.py` | 25KB | 导入导出管理 |
| **SmartConfig** | `core/smart_config.py` | 26KB | 智能环境适配 |
| **CommissionConfig** | `commission/config_manager.py` | 22KB | 佣金系统配置 |
| **DecentralizedConfig** | `decentralized_knowledge/config_manager.py` | 12KB | 去中心化配置 |
| **ProviderConfig** | `provider/config_manager.py` | 16KB | 模型槽配置 |
| **EmailConfig** | `email_notification/config_manager.py` | 5KB | 邮件配置 |

### 1.2 核心问题

1. **职责不清**: 多个配置管理器功能重叠
2. **配置分散**: 业务配置散落在各处
3. **难以统一管理**: 无法从一个入口查看/修改所有配置
4. **代码冗余**: 大量重复的配置加载/保存逻辑

## 五、API 统一

### 5.1 新增 API

```python
# core/config/unified_config.py

class UnifiedConfig:
    # ── 业务配置 ──
    
    def get_commission_config(self) -> Dict[str, Any]:
        """获取佣金系统完整配置"""
        return self.get("commission", {})
    
    def get_module_config(self, module_name: str) -> Dict[str, Any]:
        """获取佣金模块配置"""
        return self.get(f"commission.modules.{module_name}", {})
    
    def get_payment_config(self, provider: str = None) -> Dict[str, Any]:
        """获取支付配置"""
        if provider:
            return self.get(f"commission.payment.{provider}", {})
        return self.get("commission.payment", {})
    
    # ── 去中心化配置 ──
    
    def get_decentralized_config(self) -> Dict[str, Any]:
        """获取去中心化系统配置"""
        return self.get("decentralized", {})
    
    def get_p2p_config(self) -> Dict[str, Any]:
        """获取P2P配置"""
        return self.get("decentralized.p2p", {})
    
    def get_relay_config(self) -> Dict[str, Any]:
        """获取中继服务器配置"""
        return self.get("decentralized.relay", {})
    
    def get_sync_config_v2(self) -> Dict[str, Any]:
        """获取同步配置 (V2版本)"""
        return self.get("decentralized.sync", {})
    
    # ── Provider 配置 ──
    
    def get_provider_config(self) -> Dict[str, Any]:
        """获取Provider完整配置"""
        return self.get("provider", {})
    
    def get_provider_slots(self) -> List[Dict[str, Any]]:
        """获取所有模型槽位"""
        return self.get("provider.slots", [])
    
    def get_default_mode(self) -> str:
        """获取默认驱动模式"""
        return self.get("provider.default_mode", "local_service")
    
    # ── 邮件配置 ──
    
    def get_email_config(self) -> Dict[str, Any]:
        """获取邮件配置"""
        return self.get("email", {})
    
    def get_smtp_config(self) -> Dict[str, Any]:
        """获取SMTP配置"""
        return self.get("email.smtp", {})
    
    # ── SmartConfig ──
    
    def get_smart_profile(self) -> str:
        """获取当前配置方案"""
        return self.get("smart_config.profile", "default")
    
    def set_smart_profile(self, profile: str) -> None:
        """设置配置方案"""
        self.set("smart_config.profile", profile)
    
    def get_profile_config(self, profile: str = None) -> Dict[str, Any]:
        """获取指定配置方案"""
        if profile is None:
            profile = self.get_smart_profile()
        return self.get(f"smart_config.profiles.{profile}", {})
```

### 7.1 风险评估

| 风险 | 影响 | 缓解措施 |

| 迁移过程中配置丢失 | 高 | 先备份，再迁移 |

2. **兼容层保留**: 旧API通过兼容层转发到新API
3. **版本标记**: 配置文件中添加版本号，便于识别

## 九、后续优化

### 9.1 热更新支持

```python
class UnifiedConfig:
    def __init__(self):
        self._watchers: List[Callable] = []
    
    def watch(self, key: str, callback: Callable):
        """监听配置变更"""
        self._watchers.append((key, callback))
    
    def set(self, key: str, value: Any):
        """设置并通知"""
        old = self.get(key)
        super().set(key, value)
        self._notify_watchers(key, old, value)
```
