### Scrapling 0.4.7 真实 API（与假设完全不同）

| 假设（错误） | 真实 API |
|---|---|
| `Fetcher(auto_adapt=True, timeout=10)` | `Fetcher.configure(adaptive=True)`（类方法，返回 None） |
| `response.page.find()` | `response.find()`（直接在 Response 上调用） |
| `response.status_code` | `response.status` |
| `elem.markdown` | `elem.get_all_text()`（`.text` 可能为空！） |
| `elem.get('class')` | `elem.attrs.get('class', [])`（`get()` 只接受 1 个参数） |

- `_extract_content()` 用 `get_all_text()` 代替 `.text`
- 新增 `proxy` 参数支持
- `CrawlResult.content` 正确赋值（之前始终为空）

#### 2. `web_crawler/adaptive_parser.py`（重写）

#### 3. `web_content_extractor/jina_reader.py`

## ContentExtractor 自动代理接入（2026-04-26 晚）

### 背景
用户提醒：项目已有自动抓取代理功能（proxy_search 模块），Jina Reader 的代理应该自动从代理池获取，无需手动配置。

### 实现方案

#### 1. ExtractionConfig 新增自动代理
```python
@dataclass
class ExtractionConfig:
    proxy: Optional[str] = None        # 手动指定（优先级最高）
    proxy_auto: bool = True           # 是否自动从代理池获取（默认 True）

    def get_proxy(self) -> Optional[str]:
        # 优先级：手动 > 自动代理池 > None
        if self.proxy:
            return self.proxy
        if self.proxy_auto:
            pooled = get_proxy_pool().get_proxy()
            if pooled:
                return pooled.proxy.full_address  # http://ip:port
        return None
```

#### 2. JinaReader.extract() 支持动态代理
```python
async def extract(self, url, proxy: Optional[str] = None):
    effective_proxy = proxy or self.proxy  # 动态传入优先
    if effective_proxy:
        get_kwargs["proxy"] = effective_proxy
```

#### 3. ScraplingEngine.extract() 支持动态代理
```python
async def extract(self, url, ..., proxy: Optional[str] = None):
    effective_proxy = proxy or self.proxy
    if effective_proxy:
        get_kwargs["proxy"] = effective_proxy
    response = f.get(url, **get_kwargs)
```

#### 4. ContentExtractor.extract() 自动获取代理
```python
async def extract(self, url):
    proxy = self.config.get_proxy()  # 每次提取都获取（可能不同的）代理
    # L1: Jina Reader
    result = await self._get_jina_reader().extract(url, proxy=proxy)
    # L2: Scrapling
    result = await engine.extract(url, proxy=proxy)
```

### 代理池数据结构
- `Proxy`: `ip`, `port`, `protocol`, `full_address` 属性
- `PooledProxy`: 包装 `Proxy`，加了 `failure_count`, `success_count`, `score` 等
- `get_proxy_pool().get_proxy() -> Optional[PooledProxy]`: 返回评分最高的健康代理

#### 3. StreamingThoughtExecutor（`client/src/business/streaming_thought_executor.py`）
实现流式思维执行器，这是"共脑系统"的基础组件。

**核心概念**：
- **传统方式**：思考完成 → 输出完整回答 → 执行动作
- **流式思维**：思考开始 → 实时输出思考过程 → 同时执行已确定的动作

**核心功能**：
- **流式思考**：调用LLM的流式API，实时输出思考片段（ThoughtChunk）
- **并行执行**：当解析出已确定的动作时，立即执行（ActionChunk），不用等思考完成
- **思考解析**：ThoughtParser从LLM输出中解析思考片段和动作标记
- **多种输出块**：ThoughtChunk、ActionChunk、ResultChunk、FinalChunk、ErrorChunk

**动作标记格式**：
```
ACTION: search(query="xxx")
执行动作: search query="xxx"
```action
search(...)
```
```

**使用示例**：
```python
from client.src.business.global_model_router import get_global_router
from client.src.business.streaming_thought_executor import StreamingThoughtExecutor, DefaultActionExecutor

router = get_global_router()
executor = StreamingThoughtExecutor(
    model_router=router,
    action_executor=DefaultActionExecutor().execute,
)

intent = "帮我查一下今天的天气，然后计算2+2"
context = {"user_location": "北京"}

# 流式执行
async for chunk in executor.execute_stream(intent, context):
    chunk_type = chunk.get("type")
    
    if chunk_type == "thought":
        print(f"[思考] {chunk['content']}")
    
    elif chunk_type == "action":
        if chunk["status"] == "running":
            print(f"[动作] 执行 {chunk['action_type']}...")
        elif chunk["status"] == "success":
            print(f"[动作] 完成！结果: {chunk['result']}")

# 非流式执行（收集完整结果）
result = await executor.execute_with_thought(intent, context)
print(f"完整思考: {result['full_thought']}")
print(f"执行的动作数: {len(result['actions'])}")
```

### 实施内容

#### 1. 缓存优化（LRU + TTL 分层）
**修改 `GlobalModelRouter.__init__()`**：
```python
# LRU 缓存（OrderedDict，最大 1000 条）
self._cache: OrderedDict = OrderedDict()  # {hash: response}
self._cache_timestamps: OrderedDict = OrderedDict()  # {hash: timestamp}
self._cache_max_size: int = 1000

# TTL 分层（不同 capability 不同 TTL）
self._ttl_map: Dict[ModelCapability, int] = {
    ModelCapability.CHAT: 3600,           # 聊天 1 小时
    ModelCapability.CODE_GENERATION: 86400, # 代码生成 1 天（更稳定）
    ModelCapability.TRANSLATION: 7200,     # 翻译 2 小时
    ModelCapability.SUMMARIZATION: 3600,   # 摘要 1 小时
    ModelCapability.REASONING: 300,        # 推理 5 分钟（易变）
    ModelCapability.WEB_SEARCH: 60,        # 搜索 1 分钟（实时性）
}
self._default_ttl: int = 3600
```

**修改 `_get_from_cache()` - LRU 行为**：
```python
def _get_from_cache(self, cache_key: str, capability: ModelCapability = None) -> Optional[str]:
    # 1. 检查 TTL（使用 capability 对应的 TTL）
    timestamp = self._cache_timestamps.get(cache_key, 0)
    ttl = self._default_ttl
    if time.time() - timestamp > ttl:
        # 过期，删除
        del self._cache[cache_key]
        del self._cache_timestamps[cache_key]
        return None
    
    # 2. LRU：访问时移到末尾（最新位置）
    value = self._cache.pop(cache_key)
    self._cache[cache_key] = value
    self._cache_timestamps.pop(cache_key)
    self._cache_timestamps[cache_key] = time.time()
    
    return value
```

**修改 `_save_to_cache()` - LRU 行为**：
```python
def _save_to_cache(self, cache_key: str, response: str):
    # 1. 如果已存在，先删除（更新位置）
    if cache_key in self._cache:
        del self._cache[cache_key]
        del self._cache_timestamps[cache_key]
    
    # 2. 新增到末尾（最新位置）
    self._cache[cache_key] = response
    self._cache_timestamps[cache_key] = time.time()
    
    # 3. LRU 淘汰：如果超出容量，删除最旧的（队首）
    while len(self._cache) > self._cache_max_size:
        oldest_key = next(iter(self._cache))
        del self._cache[oldest_key]
        del self._cache_timestamps[oldest_key]
```

#### 2. 并发 Race 模式
**新增 `call_model_race()` 方法**：
```python
async def call_model_race(self, 
                          capability: ModelCapability,
                          prompt: str,
                          system_prompt: str = "",
                          strategy: RoutingStrategy = RoutingStrategy.AUTO,
                          context_length: int = 0,
                          top_n: int = 3,
                          timeout: float = 60.0,
                          **kwargs) -> str:
    """
    并发 Race 模式：同时调用多个模型，取第一个成功返回的
    
    Args:
        top_n: 同时调用的模型数量（默认 3 个）
        timeout: 总超时时间（秒）
    """
    # 1. 获取候选模型列表
    candidates = self.route_with_fallback(capability, strategy, context_length)
    if not candidates:
        return ""
    
    # 2. 取前 top_n 个模型
    selected = candidates[:top_n]
    
    # 3. 为每个模型创建调用任务
    async def _race_call(model: ModelInfo) -> Tuple[ModelInfo, Optional[str], bool]:
        # ... 调用模型 ...
        return model, response, success
    
    # 4. 创建所有任务
    tasks = [asyncio.create_task(_race_call(model)) for model in selected]
    
    # 5. 等待第一个成功返回
    done = set()
    pending = set(tasks)
    
    while pending:
        done_part, pending = await asyncio.wait(
            pending, 
            timeout=timeout / len(tasks),
            return_when=asyncio.FIRST_COMPLETED
        )
        
        # 检查完成的任务
        for task in done_part:
            model, response, success = task.result()
            if success:
                # 成功！取消其他任务
                for t in pending:
                    t.cancel()
                return response
        
        # 如果所有任务都失败了
        done.update(done_part)
        if len(done) == len(tasks):
            return ""
```

**新增 `start_keepalive()` 方法**：
```python
def start_keepalive(self, interval: int = 300):
    """
    启动后台心跳任务（防止模型被卸载）
    
    Args:
        interval: 心跳间隔（秒），默认 300 秒（5 分钟）
    """
    import threading
    
    def _keepalive_loop():
        """心跳循环"""
        import asyncio
        
        # 创建新的事件循环（因为在新线程中）
        loop = asyncio.new_event_loop()
        asyncio.set_event_loop(loop)
        
        while True:
            try:
                # 发送心跳请求（空 prompt，keep_alive=-1）
                for model in self.models.values():
                    if model.backend == ModelBackend.OLLAMA:
                        try:
                            loop.run_until_complete(
                                self._call_ollama(
                                    model,
                                    prompt="",
                                    system_prompt="",
                                    keep_alive=-1
                                )
                            )
                        except Exception as e:
                            logger.warning(f"心跳失败 {model.name}: {e}")
                
                # 等待 interval 秒
                time.sleep(interval)
            
            except Exception as e:
                logger.error(f"心跳循环异常: {e}")
                time.sleep(60)
        
    # 启动后台线程
    thread = threading.Thread(target=_keepalive_loop, daemon=True)
    thread.start()
    logger.info(f"心跳任务已启动（间隔 {interval} 秒）")
    return thread
```
