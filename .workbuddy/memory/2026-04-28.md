### 六类错别字支持

| 类型 | 说明 | 示例 |
|------|------|------|
| **同音错别字** | 发音相同，用字错误 | "我想吃平果" → "苹果" |
| **形近错别字** | 字形相似，看错写错 | "冒子" → "帽子" |
| **语法错别字** | 词语搭配错误 | "认直" → "认真" |

| **乱序错别字** ⭐ 新增 | 字或词的顺序颠倒 | "天今" → "今天" |

   - temperature 从 0.3 降至 0.2，提高准确性

2. **新增常量**：

## SpellCheckTextEdit 改为真正异步 ✅ (03:00)

### 文件重写：`client/src/presentation/components/spell_check_edit.py`

### 核心改进：threading.Thread → QThread + signal/slot

| 对比项 | 旧版（threading.Thread） | 新版（QThread） |
|--------|--------------------------|-------------------|
| 线程模型 | `threading.Thread` | `QThread` |
| 结果回传 | 轮询 `_check_thread_done()` | `finished` 信号（QueuedConnection） |
| UI 阻塞 | 最终降级为同步调用 | 零阻塞 |
| 线程清理 | `join()` | `quit()` + `wait()` + `deleteLater()` |
| 进度反馈 | 无 | `progress(int, str)` 信号 |

### 新版架构

```
用户输入
    ↓
_debounce_timer 超时（500ms）
    ↓
_check_async(text)
    ↓
创建 SpellCheckWorker(QThread) + Worker(QObject)
    ↓
worker.moveToThread(thread)
    ↓
连接信号：
  worker.finished  → _on_worker_finished  (QueuedConnection)
  worker.error     → _on_worker_error      (QueuedConnection)
  worker.progress  → _on_worker_progress   (QueuedConnection)
  thread.finished → worker.deleteLater()
  thread.finished → thread.deleteLater()
    ↓
thread.start()
    ↓
[工作线程] worker.run()
    ↓
调用 TextCorrectionTool.correct_text()
    ↓
emit finished(result)  → 回到主线程
    ↓
_on_worker_finished(result)
    ↓
_apply_underlines() → 红色下划线
    ↓
corrections_found.emit(corrections)
```

### 关键代码改动

**旧版**（有问题）：
```python
# 创建线程
self._worker_thread = threading.Thread(target=self._worker.run, daemon=True)
self._worker_thread.start()

# 轮询结果（阻塞 UI 风险）
self._check_timer = QTimer()
self._check_timer.timeout.connect(lambda: self._check_thread_done(text))
```

**新版**（正确）：
```python
# 创建 QThread + Worker
self._worker = SpellCheckWorker()
self._worker_thread = QThread()
self._worker.moveToThread(self._worker_thread)

# 连接信号（自动队列化）
self._worker.finished.connect(self._on_worker_finished, Qt.ConnectionType.QueuedConnection)
self._worker_thread.started.connect(self._worker.run)

# 启动线程
self._worker_thread.start()
```

## 文档→Skill 自动提炼系统 ✅ (03:45)

### 通用设计（任意文体·语义分析）

系统支持用户上传**任意文体**的文档（通知/通报/专家意见/报告/邮件…），不限制固定类型，通过 LLM 语义分析自动识别文档类型并提炼 Skill。

### 核心模块（4 个文件）

| 文件 | 功能 |
|------|------|
| `client/src/business/document_skill_extractor.py` | 文档→Skill 提炼器（LLM 语义分析） |
| `client/src/business/semantic_skill_matcher.py` | 语义匹配器（Embedding 余弦相似度） |
| `client/src/business/tools/document_skill_tool.py` | BaseTool 集成（注册到 ToolRegistry） |
| `client/src/presentation/components/document_skill_panel.py` | UI 管理面板（PyQt6） |

### 工作流程

```
用户上传/粘贴文档
    ↓
[DocumentSkillExtractor] LLM 语义分析
    ├─ 自动识别文档类型（非硬编码）
    ├─ 提炼：核心检查规则、输出模板、工作流程
    └─ 生成：SKILL.md + embedding.json
    ↓
保存到 .livingtree/skills/{skill_name}/
    ↓
下次新文档输入
    ↓
[SemanticSkillMatcher] 计算文档 embedding
    ↓
与所有已注册 Skill 的 embedding 做余弦相似度
    ↓
similarity > 0.75 → 自动触发对应 Skill
similarity ≤ 0.75 → 提示用户是否创建新 Skill
```

### DocumentSkillExtractor 核心能力

| 方法 | 说明 |
|------|------|
| `extract_skill()` | 完整流程：分析→生成 SKILL.md→保存 embedding |
| `analyze_only()` | 仅分析，不保存（预览用） |
| `_build_analysis_prompt()` | 构造 LLM 提示词（要求输出 JSON） |
| `_get_embedding()` | 调用 Ollama API 计算文档向量 |

**LLM 提示词设计**：
- 要求 LLM 自动识别文档类型（不硬编码）
- 输出严格 JSON，包含：`document_type`, `core_rules`, `output_template`, `workflow`, `faq`, `semantic_triggers`, `auto_description`
- `semantic_triggers` 用于后续 embedding 匹配

### SemanticSkillMatcher 核心能力

| 方法 | 说明 |
|------|------|
| `match()` | 返回 top-k 个最相似的 Skill |
| `match_best()` | 返回最相似的一个（或 None） |
| `auto_trigger()` | 自动触发判断（返回是否触发+消息） |
| `_load_all_skill_embeddings()` | 扫描 .livingtree/skills/ 加载所有 embedding |
| `_cosine_similarity()` | 余弦相似度计算 |

**匹配阈值**：`SIMILARITY_THRESHOLD = 0.75`（可调整）

### DocumentSkillTool（BaseTool）

注册到 ToolRegistry，支持 5 种 action：

| action | 说明 |
|--------|------|
| `extract` | 从文档提炼 Skill |
| `match` | 对文档做语义匹配，返回结果列表 |
| `auto_trigger` | 自动触发判断 |
| `list` | 列出所有已提炼的 Skill |
| `delete` | 删除指定 Skill |

### UI 面板（DocumentSkillPanel）

功能：
1. 粘贴/输入文档内容
2. 点击「🔍 语义匹配检查」→ 后台线程匹配
3. 匹配成功 → 显示匹配结果列表（含相似度）
4. 无匹配 → 点击「✨ 提炼为新 Skill」→ 输入名称 → 生成
5. 管理已有 Skill（列表 + 删除按钮）

**线程安全**：匹配和提炼操作在 `QThread` 后台执行，UI 不阻塞。

| 对比维度 | 统一大模型 | LTAI 细胞分裂再生模式 |

| 专业度 | 通用但不够专业 | 每个细胞专注一个领域 |
| 进化能力 | 重新训练整个模型 | 细胞可以复制、分裂、对抗、吞噬 |
| 共享能力 | 无法共享 | 细胞可通过 P2P 上传下载共享 |
| 替代策略 | 一次性替代 | 逐步替代（L0→L1-L3，保留 L4） |

def save_compressed_checkpoint(checkpoint: dict, path: str):
    """保存压缩的 checkpoint（比纯 pickle 小 30-50%）"""
    import tempfile
    temp_path = path + '.tmp'
    
    # 先保存为临时文件（使用 HIGHEST_PROTOCOL 减少大小）
    with open(temp_path, 'wb') as f:
        pickle.dump(checkpoint, f, pickle.HIGHEST_PROTOCOL)
    
    # 压缩为 zip
    zip_path = path.replace('.pt', '.zip')
    with zipfile.ZipFile(zip_path, 'w', zipfile.ZIP_DEFLATED) as zf:
        zf.write(temp_path, arcname='checkpoint.pt')
    
    os.remove(temp_path)

def split_cell(general_cell: str, domain1: str, domain2: str) -> bool:
    """分裂：一个通用 cell → 两个专用 cell"""
    # 1. 加载通用 cell
    model = load_cell(general_cell)
    
    # 2. 用领域1数据 fine-tune → domain1_cell
    train_domain_specific(model, domain1_data)
    save_cell(model, domain1)
    
    # 3. 重新加载通用 cell
    model = load_cell(general_cell)
    
    # 4. 用领域2数据 fine-tune → domain2_cell
    train_domain_specific(model, domain2_data)
    save_cell(model, domain2)
    
    return True
```

def adversarial_competition(cell_a: str, cell_b: str, val_data: Dataset) -> str:
    """对抗：两个 cell 在同一验证集上竞争，损失更低的胜出"""
    loss_a = evaluate_cell(cell_a, val_data)
    loss_b = evaluate_cell(cell_b, val_data)
    
    if loss_a < loss_b:
        print(f"[Adversarial] {cell_a} 胜出！（loss: {loss_a:.4f} vs {loss_b:.4f}）")
        return cell_a
    else:
        print(f"[Adversarial] {cell_b} 胜出！（loss: {loss_b:.4f} vs {loss_a:.4f}）")
        return cell_b
```

```python
def absorb_cell(teacher_cell: str, student_cell: str) -> bool:
    """吞噬：student 吸收 teacher 的知识（知识蒸馏）"""
    # 使用 L4（DeepSeek）作为 teacher，蒸馏到 student cell
    from client.src.business.global_model_router import GlobalModelRouter
    router = GlobalModelRouter.get_instance()
    
    # 蒸馏训练循环
    for batch in train_loader:
        # Teacher 预测（L4）
        with torch.no_grad():
            teacher_logits = router.call_model_sync(
                capability=ModelCapability.REASONING,
                prompt=batch['input'],
                return_logits=True
            )
        
        # Student 预测（cell）
        student_logits = student_cell_model(batch['input'])
        
        # 蒸馏损失（KL 散度）
        loss = kl_div_loss(student_logits, teacher_logits)
        
        # 反向传播
        loss.backward()
        optimizer.step()
    
    print(f"[Absorb] {student_cell} 吸收了 {teacher_cell} 的知识！")
    return True
```
