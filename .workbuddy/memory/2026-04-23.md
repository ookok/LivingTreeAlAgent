#### 2. 工具注册到 HermesAgent 的正确方式
- **不要**调用 `register_integrated_tools_to_hermes()`（那是旧接口）
- 直接用 `ToolRegistry.register()` 注册工具（与 HermesAgent._register_tools 并行）
- 工具通过 `agent.dispatcher.dispatch(name, args)` 内部调用
- HermesAgent.send_message() → LLM 解析意图 → dispatcher.dispatch() → 工具执行

#### 3. SessionDB 事务错误根因 & 修复
- **症状**: `cannot start a transaction within a transaction`
- **根因**: SessionDB 使用线程本地连接（WAL 模式），`_retry_write()` 嵌套调用 `append_message()`
- **修复方案**:
  ```python
  # 在 except 中重置线程本地连接
  agent.session_db._local.conn = None
  new_id = agent.session_db.create_session(model="qwen2.5:1.5b")
  agent.session_id = new_id
  ```
- **残留 Python 进程**会导致 `sessions.db` 被锁，杀掉进程后恢复正常

#### 4. TTS 朗读依赖状态
- pywin32 未安装 → Windows SAPI 不可用
- edge-tts 未安装 → edge-tts 后备方案不可用
- 解决方案：安装 `pip install pywin32` 或 `pip install edge-tts`

#### 5. HermesAgent SessionDB 隔离方案
- 默认路径: `C:\Users\Administrator\.hermes-desktop\sessions.db`
- 测试隔离: 使用独立路径 `PROJECT_ROOT + "/.tmp_agent_chat_db"`
- WAL 锁文件: `sessions.db-wal`, `sessions.db-shm`（进程退出后残留）
- 清理命令: `PRAGMA wal_checkpoint(TRUNCATE)` + 删除 -wal/-shm 文件

### 2. TTS 根因分析与修复
**根因（3层）：**
1. `pywin32 COM` 未注册 → 运行 `pywin32_postinstall.py -install` 后解决
2. `SoundEngine.play_voice_cn` 所有异常被静默吞掉（`except: pass`）→ 加打印
3. **最关键**：SoundEngine 继承 QObject，创建时需 QApplication；非 Qt 环境直接失败

**修复（双重保障）：**
- `SoundEngine.play_voice_cn`: WAV 文件 + winsound.PlaySound 播放（最可靠）
- `agent_chat._tts_speak_direct()`: 独立函数，直接调 SAPI + winsound
- `_tts_speak()`: 优先 SoundEngine → 降级 _tts_speak_direct

**SAPI WAV 生成正确 API：**
```python
file_stream = win32com.client.Dispatch("SAPI.SpFileStream")
audio_format = win32com.client.Dispatch("SAPI.SpAudioFormat")
audio_format.Type = 11  # SAFT16kHz16BitMono
file_stream.Format = audio_format
file_stream.Open(wav_path, 3)
speaker.AudioOutputStream = file_stream
speaker.Speak(text)
file_stream.Close()
speaker.AudioOutputStream = None
winsound.PlaySound(wav_path, winsound.SND_FILENAME)
```
