## 1. 快速开始

### 1.1 启动应用

运行 `run.bat` 或执行:
```bash
python main.py
```

**启动叙事**: "生命之树正在苏醒，根系开始向远方延伸..."

### 1.2 首次运行

首次运行会自动：
1. 检测硬件环境
2. 下载默认模型 (qwen2.5:0.5b)
3. 初始化数据库
4. 显示主界面

### 1.3 选择模型

在右侧面板的 **模型池** 中选择：
- **本地模型**: 使用 Ollama 本地模型
- **云端模型**: 使用远程 API

- **交互**:

## 4. 根系装配园 (Root Assembly Garden)

**入口**: 点击底部 Tab "🌱 嫁接园"

> 又称"嫁接外来能力的苗圃"，让开源库像外来种一样在温床中培育，然后嫁接到生命之树。

### 4.1 七阶嫁接流程

```
🌱 良种搜寻 → 🔍 林冠雷达 → ⚠️ 亲和试验 → 👨‍🌾 园丁裁决
    ↓
📦 育苗温床 → 🧪 萌芽试炼 → 🌿 扎根部署
```

### 4.2 使用方式

1. **描述需求**: 在输入框中描述您需要的功能
   - 文字描述："能解析 PDF 表格的 Python 库"
   - 或直接粘贴仓库 URL

2. **等待嫁接**: 系统自动完成七阶流程
   - 良种搜寻：解析需求
   - 林冠雷达：搜索 GitHub/Gitee
   - 亲和试验：检测冲突

3. **园丁裁决**: 如有冲突，选择处理策略
   - ✅ 并行共存：新苗与旧枝共生
   - 🔄 替换旧版：新苗替换旧枝
   - ❌ 取消：放弃此次嫁接

### 4.3 安全约束

| 规则 | 说明 |
|------|------|
| **协议白名单** | 只嫁接 MIT/Apache/BSD，拒绝 GPL 传染 |
| **网络锁** | 培育期默认断网 |
| **权限笼** | 新苗无权访问主干密钥 |

## 5. 对话式闪电上架

**入口**: 舰桥中的 📸 物质重组器 或 主界面 🛒 本地市场

### 5.1 五步上架流程

```
📸 图片上传 → AI视觉分析 → 📝 商品资料生成 → 🏷️ 伪域名链接 → 🤝 内嵌购买
```

### 4.2 使用方法

1. **拖拽图片**
   - 将商品图片拖到 "物质重组器" 区域

2. **AI 分析**
   - 自动识别品类、材质、尺寸、接口
   - 显示识别结果和置信度

3. **商品资料**
   - AI 生成标题、描述、关键参数
   - 用户补充价格和数量

4. **发布商品**
   - 生成伪域名短链
   - 广播到 P2P 网络

5. **交易履约**
   - 买家通过链接购买
   - 卖家发货确认
   - 双方互评

### 4.3 伪域名商品链接

商品发布后生成：
```
product.node8848.tree/item_abc123
```

点击链接查看商品详情页。

### 4.5 交货方式

## 7. 去中心化邮箱

**入口**: 📧 去中心化邮箱 Tab

### 6.1 地址格式

```
用户名@节点ID.p2p
示例: alice@node8848.p2p
```

### 6.2 功能

- **发送邮件**: P2P 直接投递
- **接收邮件**: 中继同步
- **外部邮箱**: 绑定 IMAP/SMTP
- **统一收件箱**: 整合所有邮件

### 6.3 安全特性

- 端到端加密 (ECDH + AES-256-GCM)
- 附件分片加密
- 信任级别管理
- 离线消息队列

### 6.4 使用流程

1. **注册地址**: 自动生成 `user@nodeid.p2p`
2. **添加联系人**: 通过节点ID添加
3. **发送邮件**: 选择联系人，撰写内容
4. **接收邮件**: 自动同步或手动拉取

## 常见问题

### Q1: 模型加载失败？

检查：
1. Ollama 服务是否运行
2. 模型文件是否完整
3. 硬件是否满足要求

### Q2: P2P 连接不稳定？

检查：
1. 网络防火墙设置
2. NAT 类型是否支持穿透
3. 中继服务器是否可用

### Q3: 如何导出数据？

在设置中：
1. 打开配置管理
2. 选择导出格式 (JSON/ZIP)
3. 指定保存路径
