## 🏗️ 架构设计

### 整体架构（3层架构）

```
LivingTreeAI/
├── client/src/              # 客户端（核心代码）
│   ├── main.py              # 入口文件
│   ├── business/            # 业务逻辑层（340+ 模块）
│   ├── presentation/        # 表现层（200+ UI组件）
│   ├── infrastructure/      # 基础设施层（DB/配置/网络）
│   └── shared/              # 共享工具类
├── server/                  # 服务端
│   ├── relay_server/        # 中继服务器（FastAPI）
│   └── tracker_server.py   # P2P追踪服务器
├── app/                     # 独立企业应用
├── mobile/                  # 移动端/PWA
├── packages/                # 共享库
└── tests/                  # 测试套件
```

### 分层架构详解

#### 1. 业务逻辑层 (`client/src/business/`)

**职责**: 所有业务逻辑、算法、数据处理

**核心模块分类**:

| 模块类别 | 主要功能 | 代表模块 |
|---------|---------|---------|
| **多智能体系统** | 智能体协作、任务分解 | `hermes_agent/`, `multi_agent/` |
| **调度系统** | 双向调度、任务编排 | `amphiloop/` |
| **优化系统** | 上下文优化、PRISM | `optimization/` |
| **企业级功能** | P2P存储、任务调度 | `enterprise/` |
| **数字孪生** | 数字化身、虚拟会议 | `digital_twin/` |
| **信用经济** | 积分系统、电商 | `credit_economy/`, `decommerce/` |
| **核心AI功能** | 语音、浏览器、会议 | `living_tree_ai/` (300+ files) |
| **检索系统** | 多源检索、知识图谱 | `fusion_rag/`, `knowledge_graph/` |
| **插件框架** | 插件管理、技能系统 | `plugin_framework/` |
| **P2P网络** | P2P网络通信 | `p2p_*` |
| **个人模式** | 个性化配置 | `personal_mode/` |
| **进化系统** | 社区进化、智能提示 | `evolving_community/`, `intelligent_hints/` |
| **办公自动化** | Office集成 | `office_automation/` |

**配置系统**:
- **新配置系统** (推荐): `nanochat_config.py` (NanochatConfig, dataclass风格)
- **兼容层**: `config.py` (UnifiedConfig, 向后兼容)

#### 2. 表现层 (`client/src/presentation/`)

**职责**: 所有UI组件、用户界面

**子目录结构**:

| 目录 | 功能 | 文件数 |
|------|------|--------|
| `panels/` | 主面板（102+ 文件） | 102+ |
| `components/` | 可复用UI组件（卡片、仪表盘等） | 50+ |
| `widgets/` | 自定义控件 | 30+ |
| `dialogs/` | 对话框窗口 | 20+ |
| `modules/` | 子模块（a2ui, connector, forum等） | 50+ |

**关键面板**:
- `enterprise_cockpit_panel.py` - 企业驾驶舱
- `finance_hub_panel.py` - 金融面板
- `game_hub_panel.py` - 游戏面板
- `evolution_panel.py` - 进化引擎面板
- `marketplace_panel.py` - 生态市场面板

#### 3. 基础设施层 (`client/src/infrastructure/`)

**职责**: 数据库、配置、网络、模型、存储

**核心组件**:
- **数据库迁移**: `database/` (v1-v14)
- **配置管理**: 读取配置、环境变量
- **网络通信**: HTTP客户端、WebSocket
- **模型管理**: 模型加载、推理
- **存储管理**: 文件存储、缓存

#### 4. 共享层 (`client/src/shared/`)

**职责**: 工具类、辅助函数、常量
