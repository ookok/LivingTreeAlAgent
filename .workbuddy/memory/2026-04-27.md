## 下午：自我进化引擎 + 系统设计理念

### 用户补充的重要理念
用户补充了非常关键的设计理念：

1. **系统要"活"起来**：
   - 目前系统不具备自我进化能力（工具方法都是"死"的）
   - 智能体的大脑（LLM）可以不断升级，但工具方法不能是死的
   - 需要自主发现缺失功能、自主学习、自主创建工具、自主完善功能

2. **代理源自动管理**：
   - 如果有外部工具或开源代码（访问不了，要思考是不是使用代理源）

3. **CLI 工具自动发现和封装**：
   - 可以使用 CLI anything 创建 CLI 工具，作为自己的武器库
   - 学习使用，例如环评的计算模型可以通过 CLI 调用
   - 以后很多软件都会提供 CLI

4. **模型自动检测与升级**：
   - 在用户提供了新的 LLM API 连接后
   - 能够自动判断可用模型，对比现有模型
   - 自主完成决策和升级

5. **完整的自我进化能力**：
   - 基于这样的设计，整个系统具备了自我升级迭代的能力
   - 后续都不需要介入开发了，交给系统自主实现

6. **UI 层面极简设计**：
   - 采用极简设计的理念
   - 减少固定化的流程和按钮堆砌
   - 主打会话交互，体现系统"智慧"

7. **工具自主修复与共享**：
   - 工具有问题，也要自主修复
   - 好用的工具自动共享到中继服务器，供其他智能体下载使用
   - 形成集体智慧

8. **系统是一个智能体**：
   - 每个功能模块又是活的智能体
   - 都有智慧的智能体，这是一个活的系统

9. **节能环保低成本**：
   - 兼顾效率
   - 一定要树立这样的理念

3. **智能体是大脑**：负责思考决策，工具是手脚眼睛耳朵

6. **极简 UI**：只有对话，没有按钮堆砌
7. **集体智慧**：工具共享，协同进化
8. **节能环保**：动态模型选择、模型自动休眠、结果缓存
