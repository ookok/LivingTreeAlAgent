4. **Auto-Publishing**: git add/commit/push → CI/CD 触发

### SerenaAdapter 双模式
- **ONLINE**: MCP 协议与 Serena Server 通信（find_symbol / insert_after / rename / diagnostics）
- **FALLBACK**: ASTFallbackEngine 本地 AST 解析（Python ast 模块）

### 安全机制
- dry-run 预览模式
- 文件备份（ModifiedFile 追踪）
- 语法验证（ast.parse）
- 审核回调（on_approve）
- 符号级编辑（不整文件重写）

## OpenCode 嵌入式集成 + oh-my-opencode 插件 ✅

### 新建文件（6 个）

| 文件 | 功能 |
|------|------|
| `libs/opencode_integration/opencode_manager.py` | 主集成管理器（OpenCodeCLI、插件管理、仓库同步） |
| `libs/opencode_integration/__init__.py` | 包初始化 |
| `client/src/business/opencode/opencode_bridge.py` | LivingTree 核心桥接器（非阻塞流读取、消息管道） |
| `client/src/business/opencode/__init__.py` | 包初始化 |
| `libs/sync_opencode.py` | 一键同步脚本（支持 force 模式） |
| `libs/README.md` | 安装指南 |

### 架构设计

```
LivingTree Agent
    └── OpenCodeBridge (单例)
            ├── OpenCodeCLIWrapper (进程管理)
            ├── OpenCodePluginManager (oh-my-opencode)
            └── EmbeddedRepoSyncer (git sync)
```

### 核心功能

1. **OpenCodeCLIWrapper**
   - 非阻塞流读取器 (NonBlockingReader)
   - 进程生命周期管理
   - 消息管道通信

2. **OpenCodePluginManager**
   - oh-my-opencode 构建 (Bun/npm)
   - 插件自动配置

3. **EmbeddedRepoSyncer**
   - 从 GitHub 同步 opencode-core
   - 从 GitHub 同步 oh-my-opencode
   - 支持 force 模式

### 使用方法

```python
# Python API
from client.src.business.opencode import get_bridge

bridge = get_bridge()
bridge.setup()  # 首次设置
status = bridge.check_status()
```

```bash
# 命令行
python libs/sync_opencode.py        # 同步
python libs/sync_opencode.py --force # 强制同步
```

### 注意事项

- GitHub 仓库需要手动克隆（网络不稳定）
- oh-my-opencode 构建需要 Bun 或 npm
- libs/opencode-core/ 和 libs/oh-my-opencode-plugin/ 在 .gitignore 中排除
